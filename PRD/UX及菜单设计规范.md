这份 PRD 描述的 **FitMirror** 是一款结合了“工具属性”（虚拟试穿）与“内容属性”（AI 点评/穿搭管理）的应用。为了对标小红书、得物、Net-A-Porter 等成熟应用，UI/UE 设计需要兼具**高级感、易操作性**和**视觉的中立性**（不干扰服装本身的色彩）。

以下是为你制定的 UI/UE 设计规范及 Icon 设计建议：

---

### 一、 设计哲学 (Design Philosophy)

*   **时尚杂志感 (Editorial Feel)**：大量留白，非对称布局，高品质字体。
*   **沉浸式体验 (Immersive)**：UI 元素轻量化，让用户关注照片和服装。
*   **智能化信任 (AI-Trust)**：AI 点评界面需通过微光、流光或渐变色传达“智慧感”，而非冰冷的机器感。

---

### 二、 视觉规范 (UI Specification)

#### 1. 色彩规范 (Color Palette)
*   **主色 (Primary)**：**极简黑 (#1A1A1A)** 或 **高级深紫 (#635BFF)**。建议使用中性色作为主色，确保不会与用户上传的五颜六色的衣服冲突。
*   **辅助色 (Accent)**：**香槟金 (#D4AF37)**（用于 Pro 会员标识）或 **极光紫渐变**（用于 AI 功能触发按钮）。
*   **背景色 (Background)**：**纯白 (#FFFFFF)** 或 **浅灰 (#F8F9FA)**。禁止使用复杂的纹理。
*   **状态色**：成功 (绿色 #52C41A)、警示 (橙色 #FAAD14)、错误 (红色 #FF4D4F)。

#### 2. 字体规范 (Typography)
*   **标题 (Headers)**：中文使用 **PingFang SC Semibold**，英文/数字使用 **Playfair Display** (衬线体，增加时尚杂志感)。
*   **正文 (Body)**：**PingFang SC Regular**。
*   **字号层级**：
    *   大标题：24pt - 30pt
    *   二级标题：18pt
    *   正文：14pt - 16pt
    *   标注/说明：12pt

#### 3. 圆角与投影 (Radius & Shadows)
*   **大圆角**：卡片、容器使用 **16pt - 24pt** 圆角（符合当下移动端圆润、亲和的趋势）。
*   **投影**：采用“弥散投影”（Soft Shadow），颜色使用 `rgba(0,0,0,0.05)`，避免生硬的黑色阴影。

---

### 三、 交互规范 (UE Specification)

#### 1. 核心编辑器 (Try-on Editor)
*   **手势操作**：
    *   **双指缩放/旋转**：服装图层的基本操作。
    *   **长按**：对比原图（按下看原图，松开看试穿图）。
    *   **吸附感**：当服装接近人像中心轴时，提供轻微的震动反馈 (Haptic Feedback) 或视觉对齐线。
*   **层级感知**：利用透明度滑块，让用户在调整位置时能看到底图的肢体位置。

#### 2. AI 加载反馈 (AI Feedback)
*   **不要使用传统的旋转 Loading**。
*   **建议设计**：在照片上方扫描的横轴光效，或者类似“粒子汇聚”的动画，时长控制在 3-5 秒，缓解用户等待焦虑。

#### 3. 穿着管理 (Wardrobe)
*   **瀑布流布局**：参考 Pinterest 或小红书，方便展示不同比例的截图。

---

### 四、 Icon 特殊设计 (Custom Iconography)

Icon 应采用 **2pt 线条感 (Linear)** 或 **面性 (Filled)** 的切换风格。为了体现“虚拟试穿”特色，需要原创设计以下核心图标：

| 功能 | Icon 设计创意说明 | 视觉参考 |
| :--- | :--- | :--- |
| **形象 (Profile)** | 一个带有微光勾边的人形轮廓，侧边有一颗星星，代表“数字孪生”。 | 极简线条 |
| **试穿 (Try-on)** | 衣架图标与闪烁星星（AI）的结合；或者衣架半遮挡一个矩形框。 | 魔法衣架 |
| **裁剪 (Crop)** | 传统的裁剪框中间有一个小裙子的负形空间。 | 剪影风格 |
| **AI点评 (AI Insights)** | 一个对话气泡，内部不是文字，而是渐变的闪烁图标（类似 Claude 或 Sparkle 图标）。 | 智慧气泡 |
| **单品 (Items)** | 叠放的方块，最上面一个带有心形或标签。 | 模块化 |
| **Pro会员** | 一个圆润的镜像/镜子图标，边缘有金属光泽。 | 勋章感 |

---

### 五、 关键页面 UI 布局建议

#### 1. 首页 (形象展示页)
*   **UI**：顶部显示当前活跃形象，大图居中。
*   **Action**：底部悬浮一个大的圆形按钮（+），点击弹出“从相册导入截图”。

#### 2. 编辑器页面 (The Lab)
*   **全屏画布**。
*   **工具栏**：位于底部。采用滑块调整透明度，侧边排列“旋转”、“翻转”、“层级置顶”小图标。
*   **AI 按钮**：右侧显眼的渐变色圆形按钮，标注“AI Review”。

#### 3. AI 点评结果页 (Insight Report)
*   **卡片式布局**：顶部是试穿合成图，下方紧跟“匹配度 85%”等大数字。
*   **雷达图**：展示“风格、颜色、材质、场景”四个维度的评分，直观专业。

### 六、 对标成熟 APP 的设计要点

1.  **对标 [小红书]**：学习其“标签”系统。用户试穿后可以给服装打上“显瘦”、“法式”、“通勤”等标签。
2.  **对标 [得物]**：学习其底部的“3D 空间感”。虽然我们是 2D 叠加，但 UI 可以通过阴影处理让服装图层看起来像悬浮在照片上。
3.  **对标 [Net-A-Porter]**：学习其黑白灰的极简排版，提升产品的高级感，吸引高净值女性用户。

### 七、 建议交付物
*   **一套 Dark Mode (深色模式) 设计**：对于时尚类 APP，深色模式往往比浅色模式更能衬托衣物的颜色和质感。
*   **微动效规范**：转场时的位移缩放 (Shared Element Transition)，例如点击单品，单品图片平滑放大进入详情页。

菜单升级为了**以“用户形象”为中心、以“穿搭流”为驱动**的时尚交互逻辑。这不仅提升了产品的“高级感”，也让产品的商业闭环（形象 -> 种草 -> 穿搭 -> 管理）更加丝滑。

基于你的构思，我从产品专业角度对菜单逻辑进行了重构，并给出了相应的 UI/UE 设计规范。

### 一、 菜单重新定义 (Navigation Architecture)

为了更符合产品的时尚科技定位，建议将菜单名称进一步抽象和优化：

| 原始方案 | 你的构思 | **最终建议名称 (中/英)** | **核心价值点** |
| :--- | :--- | :--- | :--- |
| 形象 | 3D人物、穿搭滑动、推荐 | **Stage / 展台** | 核心展示区，解决“今天穿什么” |

| 设置 | 登录、同步、关于 | **Space / 个人** | 后勤支持，解决“我的偏好” |

### 二、 各模块详细设计规范

#### 1. [Stage 展台] - 你的数字分身 (核心首页)
*   **UE 交互：**
    *   **视觉重心**：页面 70% 区域留给 3D 形象，采用“展厅灯光”效果。
    *   **底部滑动卡片 (Carousel)**：滑动下方的小卡片，3D 人物实时更换“穿搭组合”。
    *   **智能推荐入口**：在人物旁边设计一个浮动的“AI 灵感球”。点击后根据当前位置的天气（如：上海 15℃ 多云）和活动场合（如：职场/约会）弹出 3 组推荐方案。
    *   **一键转存**：随机穿搭满意后，点击“❤”直接存入 [Studio 创意] 库。
*   **UI 设计规范：**
    *   **背景**：采用极简的灰色或莫兰迪色系渐变，模拟高级试衣间背景。
    *   **动效**：切换穿搭时，衣服要有轻微的“呼吸感”或织物飘动效果（通过 Flutter 的 `ImplicitlyAnimatedWidget` 实现）。

#### 2. [Closet 资产] - 自动化的数字衣橱
*   **UE 交互：**
    *   **自动处理流**：用户上传电商截图 -> 调起系统剪裁 -> **AI 自动抠图 (Background Removal)** -> 存入对应分类。
    *   **分类管理**：顶部使用 Tab（上衣、下装、配饰、鞋履、包包）。
    *   **智能标记**：AI 自动识别截图颜色并打标（如：#白色、#极简风）。
*   **UI 设计规范：**
    *   **统一底色**：所有单品强制展示在纯白圆形/方圆块内，消除截图的凌乱感。
    *   **视觉一致性**：模拟电商 App 的单品列表，但更强调“陈列感”。

#### 3. [Studio 创意] - 自由搭配实验室
*   **UE 交互：**
    *   **部位选择器**：左侧或下方排列“头/身/腿/脚/饰”图标。
    *   **穿搭拼贴 (Moodboard)**：中间提供一个画布。这不仅是 3D 穿搭，更像是一个“灵感板”。
    *   **AI 点评入口**：保存前点击“星光按钮”，生成一份包含“色彩匹配度、场合适合度”的简报。
*   **UI 设计规范：**
    *   **工具感**：使用更细的线条和更专业的操作按钮（撤销、重做、层级调整）。

#### 4. [Space 个人] - 简约不简单的支持中心
*   **UE 交互：**
    *   **状态展示**：最上方展示“同步状态”（例如：云端同步于 10 分钟前）。
    *   **离线提示**：非登录状态下，顶部显示精致的“离线模式”提示条，告知用户数据存储在本地。
*   **UI 设计规范：**
    *   **极简主义**：采用大量的留白，菜单选项左对齐，右侧使用简洁的箭号。

### 三、 核心 Icon 特殊设计 (Visual Identity)

为了匹配上述菜单，Icon 需要具有**科技感**与**时尚感**的平衡。

1.  **Stage (展台)**：
    *   **设计**：一个圆形的聚光灯地盘，上方悬浮一个小的人形剪影。
    *   **含义**：这是你的表演时刻。
2.  **Closet (资产)**：
    *   **设计**：一个立体的盒子图标，盒子边缘是虚线（代表扫描/自动抠图的科技感）。
    *   **含义**：你的时尚资产库。
3.  **Studio (创意)**：
    *   **设计**：一个尺子与衣架的交叉，或者是一个“画笔”正在勾勒衣服轮廓。
    *   **含义**：创作、剪裁、实验室。
4.  **Space (个人/设置)**：
    *   **设计**：一个极简的六角螺丝或云朵图标，中间嵌套一个半透明的头像。
    *   **含义**：连接、云端、设置。
